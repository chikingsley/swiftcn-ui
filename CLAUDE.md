# swiftcn-ui agent rules

swiftcn-ui is a 1:1 SwiftUI port of shadcn/ui (plus elevenlabs-ui). Upstream
fidelity outranks local taste: treat ported sources like vendored code —
written by people who thought hard about them. Extend around them; do not
reorganize them.

## The worklist

`TODO.md` is the only project worklist. Every item has two independent gates:
`CODE` (item-by-item source audit against the current upstream) and
`VALIDATION` (exercised in real macOS and iPadOS hosts). Follow the protocol
at the top of that file; a passing checker or an existing file never checks a
box.

## Source rules

- **Do not restructure accepted sources.** File shape is fixed at `CODE`
  acceptance: no splitting, merging, or reorganizing files to satisfy tooling
  or preference. If a lint rule complains about upstream-shaped code, change
  `.swiftlint.yml`, never the code (file_length is already sized for
  upstream-scale files: warning 1300 / error 1600). Multi-file components
  (Combobox, Sidebar, Waveform, …) are only those accepted that way.
- Behavior or API changes to an accepted component must update that item's
  `TODO.md` note in the same change.
- Components read `@Environment(\.theme)` tokens only — never raw palette
  colors, never `colorScheme` branches. Spacing is hardcoded on the 4pt grid
  from the upstream component's Tailwind values (docs/architecture.md,
  "Spacing" — including the documented off-grid exceptions).
- `Sources/Swiftcn/Theme/Palette.swift` is generated by
  `scripts/generate_palette.py`; never hand-edit values.
- No Swift 6.2+-only syntax (e.g. `@MainActor` isolated conformances): CI and
  copy-source consumers build with Swift 6.1. Use `@preconcurrency`
  conformances instead.

## Build and gates

- Day-to-day build: `swift build --package-path Showcase` only — one `.build`
  compiles library + gallery. The root `swift build` and the iOS `xcodebuild`
  are pre-push verification.
- If deleting source files strands the build with "missing inputs", remove
  `Showcase/.build/build.db` and rebuild.
- Before landing, all of:
  `swift format lint --configuration .swift-format --recursive --parallel
  --strict Sources Showcase/Sources cli/Sources`;
  `swiftlint lint --strict --config .swiftlint.yml`;
  `python3 scripts/generate_registry.py` (then `--check`);
  `python3 scripts/check_shadcn_parity.py`;
  `python3 scripts/check_elevenlabs_parity.py`;
  `python3 scripts/check_concurrency_annotations.py`.
- The macOS validation host lives in `Validation/` (xcodegen; build once with
  `build-for-testing`, then `test-without-building` per run — see
  docs/architecture.md, "Building and the development loop").
